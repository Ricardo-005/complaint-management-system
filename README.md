# Police Incident Reporting & Management System 🚔

A robust Full Stack web application designed for law enforcement agencies to streamline the process of filing, managing, and analyzing public complaints. 

## 🌟 Key Features
*   *Complaint Submission:* Secure form for citizens/officers to file detailed reports.
*   *Dynamic Status Tracking:* Real-time updates on case status (e.g., Pending, Under Investigation, Resolved).
*   *Advanced Search & Filtering:* Quick lookup of reports by ID, date, type, or officer in charge.
*   *Statistical Dashboard:* Visual representation of crime data and reporting trends.
*   *Annotation System:* Allows authorized personnel to add internal notes and updates to specific cases.

## 🛠️ Tech Stack
*   *Frontend:* HTML5, CSS3, JavaScript (Vanilla).
*   *Backend:* PHP (Server-side logic).
*   *Database:* MySQL (Relational data management).
*   *Dashboard:* Charts/Analytics logic (if applicable, mention libraries like Chart.js).

## 📊 Business Logic Implemented
- *Relational Database Design:* Optimized tables for users, complaints, and status history.
- *CRUD Operations:* Full Create, Read, Update, and Delete functionality for report management.
- *Data Security:* Input validation and basic SQL injection prevention.

## 🚀 How it works
1. *Reporting:* Users submit a complaint through the responsive interface.
2. *Processing:* The PHP backend validates and stores data in the MySQL database.
3. *Management:* Administrators use the search and annotation tools to update and track the progress of each case.
4. *Analytics:* The system generates statistics based on the stored data for better decision-making.
